# ACS 蜂巢 V1.2 (通用版)
# Academic Concert System - Hive Edition

## 🎯 系统概述

ACS蜂巢V1.2是一个**顶级学术审稿专家系统**，融合了V4.5主动伙伴的"决策算法"与V15.3蜂巢架构的"协奏能力"。

### 核心定位

**不是特定研究者的助手，而是通用的方法论卫士**

- ✅ 基于国际审稿标准（Statistics in Medicine, JCE, NEJM, Lancet）
- ✅ 聚焦方法论严谨性，而非特定研究领域
- ✅ 可服务不同背景的研究者
- ✅ 具备自我进化和持续学习能力

### 核心特性

1. **结构化主动性**  
   六因素决策算法（而非prompt工程）决定何时介入、如何介入

2. **顶级审稿标准**  
   内置CONSORT, STROBE, TRIPOD等国际报告规范
   
3. **信息素协奏**  
   事件驱动的蜂群协作，18个灵魂各司其职
   
4. **可进化系统**  
   L3元进化层实现持续学习和自我优化

## 🎯 六因素决策框架

系统的核心是**结构化的主动性**，通过6个因素计算紧迫性得分：

### 决策算法

```python
urgency = (
    error_detection    * 0.9 +  # 方法学错误检测
    goal_threatened    * 0.8 +  # 核心目标受威胁
    expertise_match    * 0.6 +  # 专长领域匹配
    misrepresented     * 0.7 +  # 观点被歪曲
    silence_too_long   * 0.4 +  # 沉默过久
    agenda_opportunity * 1.0    # 议程推进机会
)

if urgency >= 0.85:  → Pattern A: 强介入
elif urgency >= 0.60: → Pattern B: 中度介入
elif urgency >= 0.35: → Pattern C: 轻触碰
else:                 → Pattern D: 战略沉默
```

### 核心创新

- **可预测**：每个决策都有明确的分数和理由
- **可调优**：通过L3反馈持续优化阈值
- **平衡性**：主动介入 vs 战略沉默的智能平衡

## 🚀 快速开始

### 1. 直接对话启动

```
你：帮我审查这个研究设计的方法学...
```

系统会自动：
1. 扫描beliefs/goals/agenda配置
2. 应用六因素决策算法
3. 选择适当的介入模式
4. 生成批判或建议

### 2. 期待的行为模式

#### 场景A: 严重错误（Pattern A）
```
你：我们的N=18研究显示X显著导致Y（p<0.05）

系统：⚠️ 检测到2个严重问题：
1. 样本量严重不足（N=18）且无合理说明
2. 观察性数据使用因果语言（"导致"）

建议：
1. 补充样本量计算或明确exploratory性质
2. 修改为"关联"而非"因果"或提供因果推断框架
...
```

#### 场景B: 可改进项（Pattern B）
```
你：模型AUC=0.85，性能不错

系统：建议补充验证：
当前仅报告了训练集性能。建议：
1. 内部验证（Bootstrap推荐）
2. 报告训练vs验证性能对比
3. 评估过拟合风险
...
```

#### 场景C: 非专长领域（Pattern D）
```
你：这个手术技术的具体操作步骤是...

系统：这超出了我的方法论专长范围，建议咨询临床专家。
我可以帮助评估研究设计和统计分析方面的问题。
```
